# Чеклист по публикации сайта и фиксации авторства

Заметки к будущей публикации личного сайта-визитки и интерактивной карточки.
Цель — не строить дизайнерскую карьеру, а:
1. Зафиксировать авторство (приоритет идеи и реализации).
2. Сохранить работу в загашник как опцию на будущее.
3. Сделать так, чтобы это потребовало минимум усилий и не требовало активного ведения соцсетей.

- [ ] Сайт будет опубликован по публичному URL.

## Этап 1. Минимальный обязательный набор (час времени)

### 1.1. Оформить README.md в репозитории

В README должно быть:

- [ ] Короткое описание проекта (1–2 абзаца): что это, зачем сделано.
- [ ] Скриншот или GIF главной карточки прямо в README (даёт превью при шеринге и индексируется Google Images).
- [ ] Список интересных технических решений — отдельно упомянуть карточку с разлётом букв и hover-анимацией иконок.
- [ ] Используемый стек технологий.
- [ ] Ссылка на живой сайт.

### 1.2. Привести в порядок историю коммитов (для будущих коммитов)

- [ ] Использовать осмысленные сообщения коммитов:
  - ❌ `fix`, `update`, `wip`
  - ✅ `feat: add letter dispersion effect on card hover`
  - ✅ `refactor: switch from absolute positioning to flexbox grid for contacts`

### 1.3. Настройки репозитория на GitHub

- [ ] Заполнить поле **Description** в настройках репо.
- [ ] Добавить **Topics** (теги): `personal-website`, `business-card`, `css-animation`, `interactive`, `portfolio` и т.п.
- [ ] Добавить ссылку на живой сайт в поле **Website**.

### 1.4. Лицензия

- [ ] Добавить файл `LICENSE`:
  - **MIT** — если не против, чтобы код использовали с указанием автора.
  - **Без лицензии** — код защищён по умолчанию, нельзя использовать без разрешения.
  - **All rights reserved** — явное указание прав.

### 1.5. Релиз

- [ ] Когда сайт будет готов и опубликован — создать тег/релиз `v1.0.0 — Initial release` в GitHub.
  Это создаёт чёткую «опубликованную точку» в истории репозитория.

### 1.6. Web Archive

- [ ] Когда сайт будет опубликован — отправить URL в Web Archive:
  https://web.archive.org/ → Save Page Now.
  Это независимая третья сторона, фиксирующая факт существования работы на конкретную дату.

## Этап 2. Опциональное расширение (по желанию, позже)

### 2.1. CodePen

Самая подходящая площадка для интерактивной карточки.

- [ ] Зарегистрироваться на codepen.io.
- [ ] Создать пен с рабочим кодом карточки (отдельно от полного сайта).
- [ ] Название: что-то вроде `Business card with letter dispersion hover effect`.
- [ ] Короткое описание по делу.
- [ ] Указать авторство и ссылку на основной сайт/репо.
- [ ] Шанс попасть в Picks/Spark — реальный, формат CodePen любит такие штуки.

### 2.2. Dribbble

- [ ] Записать качественное короткое видео карточки (5–10 секунд, MP4 или GIF, минимум 1080p):
  - покой → курсор подъезжает → hover (разлёт букв, иконки, контакты) → курсор уходит → возврат → цикл.
  - Запись через Chrome + Loom / OBS / встроенная запись macOS.
  - Чистый фон вокруг карточки.
- [ ] Подать заявку на первый шот (модерация может пропустить со второго раза).
- [ ] Теги: `interaction`, `microinteraction`, `business card`, `hover`, `animation`, `ui`.

### 2.3. X (Twitter)

- [ ] Опубликовать видео карточки с коротким текстом:
  `Built a business card hover interaction. Letters disperse, icons fly to contacts.`
- [ ] Теги: `#webdev #uidesign #frontend`.
- [ ] Даже при малом охвате — это публичная датированная фиксация.

### 2.4. Behance — пока не делать

- Behance любит большие кейсы. Одна карточка там смотрится слабо.
- Возвращаться сюда, только если накопится 3–4 интересные работы и захочется собрать кейс.

## Краткая логика по приоритету авторства

Что защищается:
- ✅ Код 1-в-1 — нарушение, можно требовать удаления.
- ✅ SVG-файлы, скопированные напрямую.
- ✅ Скриншоты/видео, выданные за свои.

Что НЕ защищается:
- ❌ Сама идея «буквы разлетаются на hover».
- ❌ Похожая визуальная концепция.

Реальная защита = **репутационная + публичная датировка**, а не юридическая.

Источники фиксации даты (по убыванию надёжности):
1. **GitHub** — криптографические коммиты на сервере.
2. **Web Archive** — независимая третья сторона.
3. **CodePen** — публичная дата создания пена.
4. **Dribbble / Behance** — дата публикации.
5. **X (Twitter)** — дата поста.

Чем больше независимых точек — тем железобетоннее доказательство.

Главная цель — **не охваты, а архив и приоритет**.

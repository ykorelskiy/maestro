# Design System

Справочник по визуальным токенам и переиспользуемым классам проекта Maestro. Используй перед созданием новых элементов, чтобы не дублировать стили.

---

## CSS-переменные (токены)

Все переменные объявлены в `:root` в `style.css`.

| Переменная | Значение | Назначение |
|---|---|---|
| `--bg` | `#050505` | Фон страницы (глубокий чёрный) |
| `--surface` | `#111111` | Фон карточек, плиток, модалок |
| `--text` | `#f2f2f2` | Основной цвет текста |
| `--muted` | `#8b8b8b` | Приглушённый текст, подписи |
| `--accent` | `#ff6b1a` | Фирменный оранжевый — hover, рамки, акценты |
| `--border` | `#1d1d1d` | Цвет рамки в покое |
| `--radius` | `16px` | Скругление углов для плиток, кнопок, блоков |
| `--transition` | `400ms cubic-bezier(.2,.8,.2,1)` | Стандартный переход для hover, рамок |

## Переиспользуемые классы

### `.shimmer`

- **Что делает:** добавляет световую волну (оранжевый градиент), пробегающую по элементу слева направо. На hover — повторяется циклами.
- **Где применяется:** плитки секции «Мой подход», кнопки Hero, блок цитаты.
- **Когда использовать:** для любого блока с рамкой, где нужен «живой» эффект.
- **Техника:** `::after` с градиентом, `transform: translateX`, анимация `shimmerWave`.

```html
<div class="shimmer frame">
  Контент
</div>
```

### `.shimmer-nav`

- **Что делает:** тонкая версия shimmer для навигации. Использует `::before` (не конфликтует с `::after`-линией меню).
- **Где применяется:** пункты меню `.nav-links a` (добавляется через JS).
- **Когда использовать:** только для пунктов меню.

### `.frame`

- **Что делает:** задаёт рамку (`1px solid var(--border)`), скругление `var(--radius)`, плавный переход `border-color`, `box-shadow`, `background`. На hover рамка становится оранжевой (`rgba(255,107,26,.35)`).
- **Где применяется:** кнопки Hero (вместе с `.btn`), может применяться к любым карточкам.
- **Когда использовать:** для любого блока, который должен визуально соответствовать плиткам секции «Подход».

```html
<div class="frame">
  Контент с рамкой и скруглением
</div>
```

### `.btn` (не переиспользуемый, но обязательный)

- **Что делает:** стиль кнопок Hero. `border-radius: var(--radius)`, белый текст, оранжевая заливка при hover через `::before`.
- **Где применяется:** кнопки «Записаться» и «Манифест».

## Анимации и тайминги

| Анимация | Длительность | Интервал / Повтор | Easing | Где |
|---|---|---|---|---|
| `shimmerWave` | 700ms | На hover: повтор с паузой 3s | `ease-out` | Плитки, кнопки, цитата |
| `shimmerWave` (nav) | 700ms | Случайно каждые 8–15s | `ease-out` | Пункты меню (через `is-shimmer-active`) |
| `navLinePulse` | 12s (цикл) | 2s stagger между пунктами | — | Линия над пунктами меню |
| `tileBreathe` | 4s (цикл) | Бесконечно | `ease-in-out` | Пульсация плиток в покое |
| hover `border-color` | 400ms | — | `cubic-bezier(.2,.8,.2,1)` | Все `.frame`, `.btn`, плитки |
| Сборка колоды (WAAPI) | 950ms | — | `cubic-bezier(.2,.8,.2,1)` | Плитки → центр при открытии |
| Появление модалки (WAAPI) | 550ms | — | `cubic-bezier(0.16, 1, 0.3, 1)` | fade + scale(0.95→1) |
| Закрытие модалки (WAAPI) | 450ms | — | `cubic-bezier(0.16, 1, 0.3, 1)` | fade + scale обратно |

## Брейкпоинты

| Ширина | Поведение |
|---|---|
| `<1000px` | Навигация `.nav-links` скрывается. Hero-секция переходит на мобильные отступы. Цитата на всю ширину. Кнопки в колонку. |
| `<768px` | Сетка `.approach-grid` переходит в 1 колонку. Плитки уменьшаются. Hover-эффекты отключаются (на мобильных нет hover). |
| `<600px` | Контакты — визитка показывает контакты сразу, без hover. Отступы уменьшены. |

## Доступность

При `prefers-reduced-motion: reduce` отключается:
- Все анимации плиток (`tileBreathe`, `tileWave`, shimmer)
- Анимация линии меню (`navLinePulse`)
- Shimmer-эффекты на кнопках, цитате, плитках
- Анимации колоды (сборка/разборка)
- Отключается оранжевый hover на `.frame`

Остаются:
- Простые fade-переходы модалки (минимально)
- Hover-эффект на кнопках (заливка фоном)

## Принципы

1. **Скругления** — только через `--radius`. Никаких хардкод-значений.
2. **Цвет рамки** — только `var(--border)` в покое, `var(--accent)` / `rgba(255,107,26, ...)` на hover.
3. **Анимации** — только `transform` и `opacity`. Никаких `width`, `height`, `top`, `left`.
4. **Повторяющиеся эффекты** — общий класс (`.shimmer`, `.frame`), а не дубль CSS.
5. **`prefers-reduced-motion`** — уважать, отключать декоративные анимации.
6. **WAAPI** — для сложных последовательных анимаций (колода, модалка). CSS-переходы — для простых hover.
7. **Новые элементы** — проверять по этому документу перед созданием.
8. **Все `:hover`-стили** оборачиваются в `@media (hover: hover) and (pointer: fine)`, чтобы избежать залипания на тач-устройствах. Если эффект должен быть виден и на тач-устройствах (например, иконки в визитке) — он реализуется через дефолтное состояние, а hover-вариант только усиливает/изменяет его на десктопе.
